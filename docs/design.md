# 設計思想

## コンセプト

佐賀県20市町の農家が毎朝見て、今日の作業を判断できる気象ダッシュボード。
一般の天気予報にはない **農業特化の指標** を提供する。

## 農業特化の指標

| 指標 | 用途 |
|---|---|
| 積算温度 | 作物の生育ステージ予測 |
| 有効積算温度 | 基準温度（10°C）以上の積算で開花・収穫時期推定 |
| ET0（参考蒸発散量） | 灌漑量の判断 |
| 水収支（降水 - ET0） | 圃場の水分状態の把握 |
| 強風日数・散布限界 | 農薬散布の可否判断 |
| 二十四節気・七十二候 | 伝統的な農事暦との対応 |
| 前年比・平年比 | 昨年/平年との比較で異常の早期発見 |

## UI 設計

### ダークモード
- 背景: `#050810`（深紺）
- 早朝や夜の確認を想定。目に優しい。

### 情報の階層
```
市町選択セレクタ（URL連動: /太良町）
    ↓
節気ヘッダー（今日の暦・月齢）
    ↓
5日間予報（直近の作業判断・候の変わり目表示）
    ↓
アラートバー（霜・強風の注意喚起）
    ↓
期間切替（1ヶ月〜1年）+ 前年比/平年比トグル
    ↓
積算カード（期間のまとめ + 前年/平年差分）
    ↓
メトリック切替（気温/日照/降水/水分/風）
    ↓
チャート（時系列の詳細 + タップで詳細ツールチップ）
```

### 前年比・平年比表示
- **トグルボタン**: 期間ボタンの左端、オレンジ(前年)/紫(平年)で ON/OFF 明示
- **チャート**: 前年=オレンジ折れ線、平年=紫折れ線（7日移動平均）
- **積算カード**: 各値の右に差分を表示
- **1年超の期間では自動無効化**（ボタンがグレーアウト）
- **データカバー率**: 前年80%未満、平年50%未満で非表示

### チャートのインタラクション
- タップ/クリックで日別詳細ツールチップを表示
- 選択バーに半透明ハイライト
- ツールチップに当日値 + 前年値（オレンジ）+ 平年値（紫）を表示
- 右端70%超でツールチップが左フリップ

### SVG チャート
- ライブラリ不使用。軽量な手書き SVG。
- viewBox `100x55` で全デバイス対応（`preserveAspectRatio="none"`）
- 棒グラフ（今年）+ 折れ線（前年/平年/ET0 overlay）
- パス文字列は配列+join で O(n) 構築

## パフォーマンス設計

### フロントエンド
- React.memo で不要な再レンダリングを防止（全主要コンポーネント）
- astronomy-engine を manualChunks で別チャンク分離（メイン 178KB + astro 50KB）
- ForecastTable の天文計算を useMemo でキャッシュ
- 60秒タイマーの影響を React.memo で局所化

### バックエンド
- /normal の集計を SQL GROUP BY に移行（Worker CPU 負荷削減）
- KV の stale-while-revalidate で予報取得のブロッキングを回避
- 静的データ (sekki, municipalities) に長期 Cache-Control
- Open-Meteo に 8秒タイムアウト + 最大2回リトライ

## 技術的判断

### Cloudflare Workers を選んだ理由
- サーバー管理ゼロ
- エッジ配信で高速
- 無料枠で十分（日数百アクセス規模）
- Cron Triggers で日次更新が完全自動化

### Hono を選んだ理由
- Cloudflare 自身が採用しているフレームワーク
- TypeScript ファースト
- Express に似た API で学習コスト低

### D1 を選んだ理由
- SQLite 互換（既存クエリがそのまま動く）
- 500 MB 無料で十分（20市町 x 5年 ≈ 5MB）

### KV をキャッシュに使う理由
- `expirationTtl` で正確な TTL 制御
- stale-while-revalidate パターンが実装可能
- 予報は全ユーザー共通データなので KV の整合性で十分
